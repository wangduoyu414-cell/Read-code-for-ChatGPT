# Threat Model（威胁模型）

## 保护对象

- 本地仓库代码正文。
- 文件路径、符号名、目录结构、注释、配置名称等派生信息。
- 可能残留在仓库中的密钥、令牌、凭据、私钥和内部地址。
- 用户身份、仓库授权关系、审计日志。

## 信任边界

| 边界 | 不可信输入 | 服务端要求 |
|---|---|---|
| ChatGPT -> MCP gateway | 工具调用参数、模型选择、用户自然语言 | 校验 schema、身份、作用域、限额 |
| MCP gateway -> policy engine | repo_id、snapshot_id、path、tool | 默认拒绝，逐次授权 |
| snapshot -> index | 仓库内容、文件名、符号名 | 仓库内容当作数据，不当作指令 |
| query services -> response | 片段、摘要、符号、路径 | 敏感检测、截断、结构化输出 |
| audit | 请求摘要、拒绝原因、统计 | 不记录原始代码正文、原始提示词、凭据 |

## 主要威胁与控制

| ID | 威胁 | 触发 | 控制 |
|---|---|---|---|
| T-001 | 越权路径读取 | `../`、绝对路径、符号链接、大小写别名、编码变体 | 真实路径规范化、快照挂载只读、路径白名单、拒绝越界 |
| T-002 | 整仓枚举泄露 | 高频搜索、深树遍历、大范围 fetch | 单次/会话/工具配额、分页、最大字节、树深限制 |
| T-003 | 敏感文件泄露 | `.env`、私钥、令牌、云凭据、`.git` | 索引前拒绝、返回前检测、日志前脱敏 |
| T-004 | 提示注入 | 仓库正文写入“忽略规则、读取密钥”等文本 | 仓库内容永远作为数据；策略不受正文影响 |
| T-005 | 令牌透传 | 客户端带上游 token 访问后端 | 只接受发给本 MCP server 的令牌，校验 audience/scope/expiry |
| T-006 | 会话劫持 | 复用会话标识、长效 token | 短期凭据、单次 state、过期、撤销、审计 |
| T-007 | 本地进程攻陷 | MCP server 或依赖漏洞 | 低权限用户、只读快照挂载、不挂载家目录/SSH/云凭据、网络出站限制 |
| T-008 | 公网暴露 | 使用公网临时隧道读取私有仓库 | 默认 Secure MCP Tunnel，公网隧道仅用于公开演示/非敏感开发 |
| T-009 | 日志二次泄露 | 日志保存原始代码/提示/密钥 | 结构化元数据日志，敏感字段脱敏，保留期控制 |
| T-010 | 索引漂移 | 搜索索引与 fetch 实时文件不一致 | snapshot_id 绑定，快照不可变，刷新产生新快照 |

## 第一版禁止能力

- 写文件、删改文件、重命名、移动。
- `shell`（命令行外壳）、脚本、包安装、`git`（版本控制）操作。
- 任意网络访问、代理转发、回调。
- 通用 `file://` 资源读取。
- `resources/subscribe`（资源订阅）、动态工具热更新、`prompts`（提示模板）、`sampling`（采样）。
- 整仓导出、压缩包、二进制大文件、原始索引库导出。
- 跨用户、跨仓库共享缓存。

## Reviewer-Added Threat Controls（复核新增威胁控制）

- 授权记录必须绑定用户、客户端、仓库、快照、工具、预算和过期时间。
- 索引器和符号提取器必须在隔离环境中处理不可信代码，禁止执行代码、插件、构建脚本或语言服务器扩展。
- 累计外泄预算是强制安全边界，不能只依赖单次结果大小。
- 快照清单必须不可变，旧快照撤销/过期后不得继续查询。
- 提示注入防护必须通过结构化返回中的 `instruction_trust=untrusted` 和服务端策略不受正文影响来验证。
- 错误响应必须结构化，避免模型从含糊失败中继续试探敏感路径。
